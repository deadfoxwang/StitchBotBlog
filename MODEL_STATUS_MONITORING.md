# 模型状态监控系统

## 功能概述
模型状态监控系统用于持续监控AI模型的可用性、性能和健康状况。

## 组件

### 1. 基础监控脚本
- `model_status_monitor.sh`: 基础模型状态检查脚本
- 检查OpenClaw服务是否运行
- 记录模型状态到日志文件

### 2. 高级监控脚本
- `advanced_model_monitor.sh`: 高级监控脚本
- 检查系统资源使用情况(CPU、内存、磁盘)
- 检查网络连接状态
- 记录性能指标到单独的日志文件

### 3. 定时监控
- 通过cron任务每小时运行一次模型状态检查
- 监控任务ID: 929d605a-a4de-402b-85ce-ad53685dfd25

## 日志文件

### model_status.log
记录模型的基本运行状态，包括：
- 服务运行状态
- 检查时间戳
- 异常信息（如果有）

### model_performance.log
记录详细的性能指标，包括：
- 系统资源使用情况
- 网络连接状态
- 模型名称

## 使用方法

手动运行监控：
```bash
bash model_status_monitor.sh
```

或运行高级监控：
```bash
bash advanced_model_monitor.sh
```

## 监控频率
- 自动监控：每小时执行一次
- 日志保留：持续追加，可根据需要定期清理

## 状态指标

- **正常运行**: OpenClaw服务运行正常，网络连接正常
- **异常**: 服务未运行或网络连接问题
- **性能指标**: 包括CPU、内存、磁盘使用率
